Rose User Guide: Suite Tutorial (cylc broadcast)

Introduction

This part of the Rose user guide walks you through using cylc broadcast.

This changes task runtime configuration in a running suite, on-the-fly.

Purpose

cylc broadcast can be used to change any [runtime] setting during a suite's run.

You could invoke it on the command line (e.g. as a manual response to a failure) or use it in a task script (e.g. as part of an auto-recovery task that fixes other tasks).

cylc broadcast can also be used to communicate between tasks, where file-based communication is not appropriate.

Example - CLI

We'll look at some example CLI usage first.

Create a new suite (or just a new directory) with a blank rose-suite.conf and a suite.rc file that looks like this:

[scheduling]
    initial cycle time = 2013010500
    final cycle time = 2013012500
    [[special tasks]]
        cold-start = startup
    [[dependencies]]
        [[[0]]]
        graph = """
            startup | print_colour[T-24] => print_colour
        """

Runtime

This sets up a standard cycling suite.

Our daily task (print_colour) will need some runtime.

Runtime excerpt

[runtime]
    [[root]]
        [[[event hooks]]]
            succeeded handler = rose suite-hook
    [[print_colour]]
        command scripting = """
            sleep 4
            echo "Today's colour is "$COLOUR""
        """
        [[[environment]]]
            COLOUR = green

Running it

This will print some html with appropriate coloured text.

Change directory to the root of the suite.

Run the suite using:

rose suite-run

cylc broadcast

Now try running cylc broadcast to change the colour. You'll need to substitude the name of your suite for $SUITE_ID. If the suite is not running on your local machine, you should add the option --host=HOSTNAME where HOSTNAME is the suite run host (look for will run on XXX in the suite run output).

cylc broadcast --host=HOSTNAME -n print_colour -s [environment]COLOUR=red $SUITE_ID

cylc broadcast continued

Try changing the colour to something else, e.g.:

cylc broadcast --host=HOSTNAME -n print_colour -s [environment]COLOUR=blue $SUITE_ID

Output

You can see the results of your changes by looking at the output viewer. Run:

rose suite-log-view

The output of the print_colour tasks should have some coloured text.
